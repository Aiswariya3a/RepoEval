## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps inside buttons, avatar border offset |
| sm | 8px | Compact element spacing, inline padding, badge padding |
| md | 16px | Default element spacing, card padding (x/y), form field padding |
| lg | 24px | Section padding, card gap from layout edges |
| xl | 32px | Layout gaps between major sections, sign-in card width offset |
| 2xl | 48px | Major section breaks, top/bottom hero padding |
| 3xl | 64px | Page-level spacing, full-page vertical centering |

Exceptions:
- **GitHub OAuth button minimum height:** 44px (accessibility touch target — exceeds 8-point scale, intentional per WCAG 2.2 SC 2.5.8)
- **Sidebar width:** 256px (16rem, falls outside 8-point scale — standard app shell convention, matches Tailwind `w-64`)

## Interactions

### Pre-Auth: Sign-In Page

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│                    ┌──────────────┐                   │
│                    │   RepoEval   │  ← App logo/brand  │
│                    └──────────────┘                   │
│                                                      │
│   AI-powered project evaluation reports in minutes   │
│                                                      │
│   ┌──────────────────────────────────────┐           │
│   │  ┌───┐  Sign in with GitHub          │           │
│   │  │   │                                │           │
│   │  └───┘                                │           │
│   └──────────────────────────────────────┘           │
│                                                      │
│   [optional: inline error message]                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

1. **Initial state:** Centered card on page (vertically + horizontally). Brand/logo at top, value proposition tagline below, "Sign in with GitHub" button with GitHub logo, subtle border and shadow on card.
2. **Click "Sign in with GitHub":** Button shows loading state (spinner or text change to "Redirecting to GitHub…"). User is redirected to `https://github.com/login/oauth/authorize?client_id=...` (backend-determined URL).
3. **Successful OAuth callback:** GitHub redirects to backend callback → backend sets httpOnly JWT cookies → backend redirects to frontend `/dashboard`. User lands on post-auth dashboard shell.
4. **OAuth error:** Backend redirects to frontend `/sign-in?error=<code>`. Frontend renders inline error message below the sign-in button based on error code param. No separate error page.
5. **Session check:** On mount, frontend calls `/api/auth/me` (or equivalent) to check for existing session. If valid, redirect to `/dashboard` immediately.

### Post-Auth: Dashboard Shell (Layout Prep)

```
┌──────────┬────────────────────────────────────────────┐
│          │                                            │
│  ┌──────┐│  Welcome to RepoEval                       │
│  │  A  ││                                            │
│  └──────┘│  Evaluate your first repository to         │
│          │  get started.                              │
│  Nav     │                                            │
│  ────────│  ┌──────────────────────────────┐          │
│  Dashboard│  │    Create Project           │          │
│          │  └──────────────────────────────┘          │
│          │                                            │
│          │                                            │
│          │                                            │
│          │                                            │
│  ────────│                                            │
│  Sign out│                                            │
└──────────┴────────────────────────────────────────────┘
```

1. **Sidebar:** Slim vertical nav (width: 256px / `w-64`), left-aligned, full viewport height. Background: card background. Contains:
   - Top: User avatar (25px) + display name (truncated)
   - Middle: Nav items (Dashboard — active for Phase 1, Projects, Settings, etc. — placeholder structure for Phase 2+)
   - Bottom: "Sign out" link
2. **Content area:** Rest of viewport. Phase 1 shows welcome empty state (heading + body + "Create Project" button). Layout structure prepares for Phase 2 (project list table/CRUD) to be dropped into this area.
3. **Sign out:** Triggers POST to backend logout endpoint → clears httpOnly cookies → redirects to `/sign-in`.
4. **Session expiry:** On 401 response from API, redirect to `/sign-in?error=expired`. Inline error: "Your session has expired. Please sign in again."

## Responsive Behavior

| Breakpoint | Sign-In Page | Dashboard Shell |
