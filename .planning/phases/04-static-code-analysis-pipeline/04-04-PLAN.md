Key integration points:

**Current project detail page (frontend/app/projects/[id]/page.tsx):**
- Fetches project data, repos, and ingestion status on mount
- Renders repo list with IngestButton, IngestionBadge, IngestionProgressPanel
- Polls ingestion status when repos have active status

**Ingestion button interaction:**
- After ingestion completes (status === "complete"), the AnalyzeButton should appear
- During analysis, IngestButton should be disabled (repo is being used)

**Data flow for analysis lifecycle in detail page:**
1. Page loads → shows repos with ingestion status
2. User clicks "Ingest" → IngestionProgressPanel shows status
3. Ingestion completes → AnalyzeButton appears
4. User clicks "Run Analysis" → AnalysisProgressPanel shows 8-stage progress
5. Analysis completes → AnalysisResultsDisplay shows scores + metrics
6. User can click "Re-analyze" → repeats from step 4

**ProjectCard (frontend/components/dashboard/project-card.tsx):**
- Currently shows `{repoCount} repos` and aggregate ingestion badge
- Add aggregate analysis status (e.g., "Analyzed" in green if all repos analyzed)

**API types from Plan 03:**

- components: IngestButton, IngestionBadge, IngestionProgressPanel, ReposField

      status: "analyzed" | "partial" | "pending" | "running" | null

        case "partial":
          return { label: "Partial", color: "text-[#F59E0B]" };
